# Plan: Replace Micro Stories with AI-Generated Graphic Novels

## Implementation Status

Implemented on 2026-05-14. New full-pipeline generations now create graphic novels for the instructional Read step; existing packs with `MicroStory` records continue to render through the legacy fallback. The implementation follows the plan with one addition: `GraphicNovelPage.layout_description` is persisted because the image prompt needs the page-composition guidance generated by the script step.

---

## Context

Before this change, the instructional flow (Primer → Story → Cloze Quiz) used an 80-150 word micro-passage as the "Story" step. New generations now replace that with a full AI-generated graphic novel — 10-12 panels across ~5 pages in landscape 16:9 format — to create a flagship feature that differentiates the app. Text is baked into images by GPT-Image-2, with target vocabulary words rendered in a visually distinct style (glowing/colored). Existing word sets keep their micro stories; only new generations produce graphic novels.

---

## 1. Data Model (new models in `backend/vocabulary/models.py`)

**Key design decision:** Each page is generated as a single image containing all its panels. The AI composes the full page layout (panel borders, sizes, gutters) within one 1792x1024 image. This means:
- Fewer API calls (5 per pack instead of 10-12)
- More dynamic/flexible panel layouts (AI controls panel sizing naturally)
- More visually cohesive pages (consistent style, lighting, flow within a page)

```python
class GraphicNovel(models.Model):
    pack = models.OneToOneField(WordPack, on_delete=models.CASCADE, related_name='graphic_novel')
    title = models.CharField(max_length=200)
    synopsis = models.TextField(help_text='Story synopsis for character/scene continuity')
    style_prompt = models.TextField(help_text='Art style directive used for all panels')
    reading_level = models.IntegerField(help_text='Lexile score')
    created_at = models.DateTimeField(auto_now_add=True)

class GraphicNovelPage(models.Model):
    novel = models.ForeignKey(GraphicNovel, on_delete=models.CASCADE, related_name='pages')
    page_number = models.IntegerField()
    image = models.ImageField(upload_to='graphic_novels/', blank=True)
    prompt_used = models.TextField(blank=True)
    panel_count = models.IntegerField(help_text='Number of panels on this page (1-4)')
    layout_description = models.TextField(blank=True, help_text='Full-page composition guidance')
    panel_descriptions = models.JSONField(default=list, help_text='Per-panel metadata for accessibility/tooltips')
    # panel_descriptions structure:
    # [
    #   {
    #     "panel_number": 1,
    #     "narration": "...",
    #     "dialogue": [{"speaker": "...", "text": "..."}],
    #     "vocab_words": ["resilient", "overcome"],
    #     "alt_text": "..."
    #   },
    #   ...
    # ]
    vocab_words_used = models.JSONField(default=list, help_text='All vocab words appearing on this page')

    class Meta:
        ordering = ['page_number']
        unique_together = ('novel', 'page_number')
```

**Why no separate Panel model:** Since the image is per-page (not per-panel), panel boundaries exist only within the image. Panel metadata (dialogue, narration, vocab words) is stored as a JSONField on the page for accessibility and tooltip purposes. This avoids an extra table and join while keeping the data we need.



Add to `GenerationJob`: `graphic_novels_created = IntegerField(default=0)`
Add to `GenerationJobLog.Step`: `GRAPHIC_NOVEL_SCRIPT`, `GRAPHIC_NOVEL_IMAGES`. `STORY_CLOZE_GEN` remains in the enum for legacy logs/manual testing but is no longer in `PIPELINE_STEP_ORDER` for new full-pipeline generation.

Migration: `0018_graphic_novel.py` (2 new tables instead of 3, plus `GenerationJob.graphic_novels_created` and the `GenerationJobLog.step` choices update)

## 2. Pipeline Changes (`generation_pipeline_service.py`)

`PIPELINE_STEP_ORDER` replaces `STORY_CLOZE_GEN` with two new steps:

```python
PIPELINE_STEP_ORDER = [
    ..., 'PRIMER_GEN',
    'GRAPHIC_NOVEL_SCRIPT',   # Step 7A: Gemini generates script + cloze
    'GRAPHIC_NOVEL_IMAGES',   # Step 7B: GPT-Image-2 generates panel images
    'CREATIVE_DIRECTION', 'IMAGE_GEN', 'PICTURE_MATCH_GEN',
]
```

**Step 7A — `_step_graphic_novel_script()`:**
- Calls Gemini with new prompt template `graphic_novel_script.txt`
- Input: pack words + definitions, text_type, target Lexile
- LLM outputs structured JSON organized by page:
  - title, synopsis, style notes
  - pages array — each page has: page_number, panel_count, and per-panel descriptions (scene, dialogue, narration, vocab words, layout guidance)
  - cloze items
- The LLM plans panel composition with the full page layout in mind (which panels are large/small, how they flow together visually)
- Creates `GraphicNovel`, `GraphicNovelPage` (without images), `ClozeItem`
- Skips packs that already have a `graphic_novel` (resume safety)

**Step 7B — `_step_graphic_novel_images()`:**
- Iterates all pages without images (~5 pages per pack)
- Composes a single image prompt per page that includes:
  - Art style directive (from `GraphicNovel.style_prompt`)
  - Number of panels and their layout on the page
  - Each panel's scene description, dialogue (in speech bubbles), narration (in caption boxes)
  - Vocab word highlighting instructions (distinct color/glow)
  - Character continuity notes from synopsis
- Calls `call_openai_image(prompt, size="1792x1024")` — one call per page
- Saves image to `GraphicNovelPage.image`
- Continues on individual page failures; fails the step only if every pending page fails
- ~5 API calls per pack (vs 10-12 in per-panel approach)

**Rationale for two steps:** Script generation is fast (~5s) and cheap. Image generation is slow (~30-50s for 5 calls) and failure-prone. Splitting them means resume after image failure doesn't re-call Gemini.

## 4. New Prompt Templates (`backend/vocabulary/prompts/`)

**`graphic_novel_script.txt`** — Gemini prompt for script generation:
- Role: graphic novel writer for K-8 students
- Constraints: 10-12 panels across ~5 pages, 1-4 panels per page
- Key instruction: "Plan each page as a complete visual composition. Decide how many panels each page has and describe how they should be arranged (e.g., one large panel with two smaller ones below, two equal panels side by side, a single splash page)."
- Vocab words must appear naturally in dialogue/narration
- Vocab words marked with highlight instructions per panel
- Also generates cloze items (same call for narrative coherence)
- Output JSON structure:

```json
{
  "title": "The Last Signal",
  "synopsis": "Maya, a young radio operator on a space station, intercepts a mysterious signal...",
  "pages": [
    {
      "page_number": 1,
      "panel_count": 2,
      "layout_description": "Two equal panels side by side. Left panel: establishing shot. Right panel: close-up.",
      "panels": [
        {
          "panel_number": 1,
          "scene_description": "Wide shot of a space station orbiting Earth at dawn...",
          "narration": "Three months into her mission, Maya had grown used to the silence.",
          "dialogue": [],
          "vocab_words": ["isolation"],
          "vocab_highlight_note": "'isolation' appears in the narration box in glowing gold"
        },
        {
          "panel_number": 2,
          "scene_description": "Close-up of Maya's face, headphones on, eyes widening...",
          "narration": "",
          "dialogue": [{"speaker": "Maya", "text": "That frequency... it's not random."}],
          "vocab_words": [],
          "vocab_highlight_note": ""
        }
      ]
    }
  ],
  "cloze_items": [...]
}
```

**`graphic_novel_page.txt`** — Image prompt template per page (replaces per-panel approach):
- Art style section (from `GraphicNovel.style_prompt`, configurable later)
- Page composition: "Create a single 1792x1024 landscape comic book page containing {panel_count} panels"
- Layout description from script (how panels are arranged)
- Each panel's scene description, dialogue in speech bubbles, narration in caption boxes
- Vocab word highlight: "Render the words {vocab_words} in glowing gold/distinct color wherever they appear in text"
- Character continuity notes from synopsis
- Format constraints: clear panel borders, readable text, no watermarks
